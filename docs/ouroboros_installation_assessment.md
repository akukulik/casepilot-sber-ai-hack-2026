## Почему выбрана именно эта версия

`v6.72.0` — верхний стабильный официальный Release на момент проверки, а не просто максимальный тег. Для него подтверждены:

1. отдельная запись в официальном GitHub Releases;
2. готовый macOS DMG;
3. флаг `prerelease: false`;
4. заявленная поддержка macOS 12+ в README и `LSMinimumSystemVersion: 12.0` в спецификации приложения;
5. успешные официальные CI jobs `full-test (macos-latest)`, `skill-smoke (macos-latest)`, `release-preflight`, `build (macos-latest, dmg)` и `release`.

Переходить к предыдущей стабильной версии оснований пока нет.

## Release notes

Release API содержит ссылку на полный changelog между `v6.71.2` и `v6.72.0`. Запись версии в официальном README описывает исправления хронологии Chat и честного отображения UI: сортировку сообщений и task cards по исходным timestamp, сохранение позиции прокрутки, корректный replay/reconnect, отображение финального ответа обычным текстом и исправление отступа заголовка на узких окнах. Из release notes не следует изменение формата установки или повышение минимальной версии macOS.

## Совместимость с этим Mac

- Компьютер: MacBook Air с Apple M3.
- Архитектура компьютера: `arm64`.
- Установленная macOS: 26.5.2.
- Минимальная macOS приложения: 12.0.
- Официальный build script поддерживает обе macOS-архитектуры: для `arm64` выбирает `aarch64-apple-darwin`, для Intel — `x86_64-apple-darwin`.
- Официальный macOS build job для этого релиза завершён успешно.

Предварительный вывод: релиз совместим с macOS и Apple Silicon. После скачивания, но до открытия DMG, SHA-256 будет сверен с GitHub Releases API; после монтирования будет дополнительно проверена фактическая архитектура бинарника внутри конкретного DMG.

## Известные риски установки и первого запуска

- В открытых официальных материалах не найдено известной критической проблемы установки или первого запуска именно `v6.72.0` на Apple Silicon.
- Общий CI run тега имеет итог `failure`, но единственный упавший job — `integration-test` внешних model providers. macOS test/build/skill-smoke и публикация Release успешны. Это зафиксировано как риск работы с провайдерами, но не как дефект DMG или первого запуска.
- README прямо предупреждает, что при первом запуске может потребоваться right-click → Open. Полностью отключать Gatekeeper не нужно.
- Onboarding wizard не позволяет завершить настройку без runnable remote key или local model source. Поэтому smoke test без подключения провайдера может проверить запуск и доступную оболочку, но не обязан позволить завершить onboarding или отправить сообщение в Chat.

## Фактический результат установки

После подтверждения пользователя, что в чате хакатона имелась в виду именно `v6.61.4`, эта версия была установлена штатным способом:

- `Ouroboros.app` скопирован из официального DMG в `/Applications`;
- установленная версия: 6.61.4;
- бинарник: нативный `arm64`;
- минимальная macOS: 12.0;
- первый запуск выполнен через Finder → right-click → Open;
- Gatekeeper не отключался;
- `sudo` и `xattr` не использовались.

Несмотря на отрицательный результат предварительной CLI-проверки code signature, macOS разрешила штатный первый запуск через предусмотренный официальной инструкцией right-click → Open. На практике приложение успешно открыло окно `Ouroboros — Setup`.

После отдельного разрешения пользователя onboarding завершён через штатный
валидатор настроек 6.61.4:
