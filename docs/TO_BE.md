# CasePilot: целевая архитектура после MVP

## Ближайший шаг

Завершить аналитику операторского цикла:

1. взять кейс;
2. получить план и подтвердить его либо перейти к ручной работе;
3. наблюдать выполнение;
4. получить evidence-bound рекомендацию;
5. append-only сохранить финальное решение сотрудника и текст ответа клиенту.

Редактирование плана, повторное планирование и продолжение остановленного
workflow перенесены за границы хакатонного MVP.

## Переход от mock к пилоту

- заменить mock-проверки read-only интеграциями с параметризованными и
  allow-listed запросами;
- вынести runtime из JSON в транзакционное хранилище;
- добавить RBAC, SSO, idempotency keys, audit retention и шифрование;
- отделить долгие экспертизы в асинхронные задания со статусами и SLA;
- добавить contract tests с песочницами банковских систем;
- провести privacy, model risk, legal и information-security review.

## Жизненный цикл сценариев

Базовый human-governed pipeline уже реализован в MVP: gap analysis, draft,
independent validation, offline replay и explicit expert approval.
Следующие пункты описывают его production-развитие.

Сценарии должны создаваться не в автономном self-evolution production-цикле,
а через управляемый pipeline:

1. кластеризация успешно закрытых кейсов;
2. генерация кандидата сценария;
3. автоматическая проверка схемы и evidence;
4. экспертное ревью;
5. публикация версии в каталоге;
6. offline evaluation и контролируемый rollout;
7. мониторинг качества и откат.

Ouroboros может помогать генерировать и анализировать кандидатов, но публикация
сценария требует человека. A2A и MCP уместны только после появления реальных
границ систем и независимых специализированных исполнителей.

## Retrieval на масштабе

Текущий topic/subtopic + BM25 + rerank подходит для синтетического MVP.
При росте каталога:

- ввести версионирование, ownership и validity period сценариев;
- добавить фильтры по продукту, причине, ограничениям и доступным данным;
- оценивать recall@k, precision@k и долю manual review;
- только при доказанной необходимости добавить hybrid/vector retrieval;
- передавать планировщику компактные top-5 сценариев, а не всю историю.

## Продуктовый интерфейс

Целевой UI — карточка кейса слева и стратегия CasePilot справа: шаги,
источники, прогресс, исключения, решение, ответ клиенту и явные действия
сотрудника. Чат остаётся вспомогательным каналом, а не основным интерфейсом.
