# Публичное демо CasePilot

Ссылка: <https://akukulik.github.io/casepilot-sber-ai-hack-2026/>

GitHub Pages работает в статическом synthetic demo mode и не использует
OpenRouter, API-ключ или реальные банковские системы.

## Доступные кейсы

| Кейс | Сценарий | Результат |
|---|---|---|
| `VAL-DC-002` | Технический отрицательный остаток | План выполнен, счёт готов к закрытию |
| `VAL-DC-013` | Задержанный реверс предавторизации | План выполнен, hold снят в mock-контуре |
| `VAL-DC-016` | Активный merchant token | План выполнен, будущие списания остановлены |
| `VAL-DC-018` | Розыск карточного возврата | План выполнен, возврат поставлен на контроль |
| `VAL-DC-004` | Ограничение без номера документа | План остановлен на шаге 2, запрошены данные |

Каждый кейс имеет собственные описание, транскрибацию, продукты, найденный
сценарий, план, пошаговое выполнение и терминальный результат.

## Как проверить

1. Выбрать кейс в левой очереди.
2. Нажать «Запустить CasePilot».
3. Проверить выбранный сценарий и план.
4. Нажать «Подтвердить».
5. Для первых четырёх кейсов посмотреть рекомендацию и ответ клиенту.
6. Для `VAL-DC-004` проверить безопасную остановку, недостающий реквизит и
   кнопку «Добавить информацию».

Статические задержки имитируют прогресс только для презентации. Полноценный
локальный режим вызывает Ouroboros и использует runtime-контракты проекта.
